PokéChess

AutoChess genre meets Pokémon.

I attempted to create a Pokémon Autobattler but stopped because of the methods I chose to use when I first created it.

What I learned:

Firstly, I shouldn't try and fully finish the art aspect of the game while I haven't done any game-play testing. The result of my weeks long playing around with Godot and creating PokéChess resulted in me just playing around with shaders and making the game look nice. If I focused on the prototyping on the game first I would have had an easier time implementing the prototype. In researching to help me fix this I stumbled on blackbox prototyping and will be using this method in the future.

Most of the time while making this game I was using assets from Pokémon Mystery Dungeon: Blue/Red Rescue Team so most of my time was spent figuring out how efficiently take all the art assets I want. It took me a couple hours trying animate the Cubone so in the future I will write a program that takes an img file and convert it in a way so I can create new animations for different Pokémon much easier.

Through trying to figure out how Game Freak animated the water and lava in the game I discovered it was through the use of shaders, which I never touched and thought was solely for 3D game dev territory. Writing a shader script that can manipulate certain colours depending on it's hex values given to it was pretty fun as I had complete control of the image, this process showed me how powerful and effective using shaders was.
